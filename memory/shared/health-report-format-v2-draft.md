## 🎯 设计理念转变

### 旧版思维
> 记录数据 → 展示数据 → 简单建议

### Whoop 思维（新版）
> **Body OS**: 数据只是输入，**生活方式改善**才是输出  
> 像 PMO 管理项目一样管理健康 —— 追踪 → 分析 → 行动 → 反馈循环

## 1️⃣ 今日状态速览 (Recovery Score)

### 视觉设计
```
┌────────────────────────────────────┐
│     🔋 恢复度评分                   │
│                                    │
│        ┌─────────┐                 │
│        │   45%   │  ← 环形进度条   │
│        │  RECOVER│                 │
│        └─────────┘                 │
│                                    │
│  🟡 黄色区域: 建议降低训练强度      │
│  今日建议: 主动恢复 / 轻度活动      │
└────────────────────────────────────┘
```

### 评分标准 (0-100%)
| 区间 | 颜色 | 状态 | 训练建议 |
|------|------|------|----------|
| 67-100% | 🟢 绿色 | 最佳 | 可承受高强度训练 |
| 34-66% | 🟡 黄色 | 一般 | 降低强度，专注技术 |
| 0-33% | 🔴 红色 | 较差 | 主动恢复或休息 |

### 计算公式（Whoop 风格）
```
Recovery Score = f(HRV, RHR, Sleep Quality, Respiratory Rate)
```

| 指标 | 权重 | 数据来源 |
|------|------|----------|
| HRV (心率变异性) | 35% | 🔴 Google Fit 暂无 / Apple Watch 有 |
| 静息心率 (RHR) | 25% | 🟡 Google Fit 部分支持 |
| 睡眠质量 | 25% | 🟢 Google Fit 有 |
| 呼吸频率 | 15% | 🔴 暂无 |

> **当前状态**: 需用替代指标估算，或提示用户接入 Apple Watch / Whoop

### Sleep Score 计算
| 维度 | 权重 | 说明 |
|------|------|------|
| 睡眠时长 | 40% | 与个体需求对比 |

| 睡眠阶段 | 25% | Deep + REM 占比 |
| 一致性 | 10% | 入睡/醒来时间与日常规律对比 |

> **当前状态**: Google Fit 提供基础睡眠数据，Deep/REM 需估算或用睡眠追踪 App

   ├─ 今晚 22:30 前强制断网
