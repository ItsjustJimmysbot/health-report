# 添加成员说明

本文档说明如何在健康报告生成工具中添加多个成员（最多3人），以便为家庭成员或团队生成多份报告。

## 配置方法

### 步骤1：修改成员数量

编辑 `scripts/generate_v5_medical_dashboard.py` 文件，找到开头的配置区域：

```python
# ==================== 多成员配置区域 ====================
# 成员数量：默认1，最多3（控制单次运行token消耗）
MEMBER_COUNT = 1  # 可选值：1, 2, 3
```

将 `MEMBER_COUNT` 修改为需要的成员数量（1-3之间）。

### 步骤2：配置成员数据路径

在同一个配置区域，填写每个成员的健康数据和运动数据路径：

```python
# 成员1：默认使用当前用户路径（无需修改）
MEMBER_1_NAME = "默认用户"
MEMBER_1_HEALTH_DIR = None   # None表示使用默认路径
MEMBER_1_WORKOUT_DIR = None  # None表示使用默认路径

# 成员2：填写第二个成员的数据路径
MEMBER_2_NAME = "成员2"
MEMBER_2_HEALTH_DIR = ""   # 填写路径，如：Path.home() / "Dropbox" / "成员2" / "Health Data"
MEMBER_2_WORKOUT_DIR = ""  # 填写路径，如：Path.home() / "Dropbox" / "成员2" / "Workout Data"

# 成员3：填写第三个成员的数据路径  
MEMBER_3_NAME = "成员3"
MEMBER_3_HEALTH_DIR = ""   # 填写路径，如：Path.home() / "Dropbox" / "成员3" / "Health Data"
MEMBER_3_WORKOUT_DIR = ""  # 填写路径，如：Path.home() / "Dropbox" / "成员3" / "Workout Data"
```

**注意**：
- `HEALTH_DIR`：指向包含 `HealthAutoExport-YYYY-MM-DD.json` 健康数据的目录
- `WORKOUT_DIR`：指向包含 `HealthAutoExport-YYYY-MM-DD.json` 运动数据的目录（可与Health相同或不同）

### 路径格式示例

```python
# macOS 示例 - Health数据
MEMBER_2_HEALTH_DIR = Path.home() / "Dropbox" / "爸爸的健康数据" / "Health Auto Export" / "Health Data"

# macOS 示例 - Workout数据（可以与Health相同或不同）
MEMBER_2_WORKOUT_DIR = Path.home() / "Dropbox" / "爸爸的健康数据" / "Health Auto Export" / "Workout Data"

# 绝对路径示例
MEMBER_2_HEALTH_DIR = "/Users/shared/Dropbox/妈妈/Health Auto Export/Health Data"
MEMBER_2_WORKOUT_DIR = "/Users/shared/Dropbox/妈妈/Health Auto Export/Workout Data"
```

## 运行多成员报告生成

### 步骤1：为每个成员准备AI分析

由于每个成员的健康数据不同，需要为每个成员分别进行AI分析。

在AI对话中：
1. 提取成员1的数据
2. 生成成员1的AI分析
3. 提取成员2的数据
4. 生成成员2的AI分析
5. （可选）提取成员3的数据
6. （可选）生成成员3的AI分析

### 步骤2：组合AI分析为JSON数组

将所有成员的AI分析组合成一个JSON数组：

```json
[
  {
    "date": "2026-02-22",
    "generated_date": "2026-02-22",
    "hrv": "成员1的HRV分析...",
    "resting_hr": "成员1的静息心率分析...",
    ...
  },
  {
    "date": "2026-02-22",
    "generated_date": "2026-02-22",
    "hrv": "成员2的HRV分析...",
    "resting_hr": "成员2的静息心率分析...",
    ...
  }
]
```

### 步骤3：运行脚本

```bash
cd /Users/jimmylu/.openclaw/workspace-health
python3 scripts/generate_v5_medical_dashboard.py 2026-02-22 < multi_member_analysis.json
```

```

```

## 限制说明

1. **成员数量上限**：最多3人，控制单次运行的token消耗
2. **数据路径**：每个成员的数据路径必须独立且可访问
3. **AI分析**：每个成员需要独立的AI分析，不能共用
4. **缓存文件**：每个成员的缓存文件以成员名称区分

## 常见问题

### Q1: 如何确认数据路径正确？

检查路径下是否存在类似 `HealthAutoExport-2026-02-22.json` 的文件：

```bash
# Health数据
ls /路径/Health\ Auto\ Export/Health\ Data/HealthAutoExport-2026-02-22.json

# Workout数据
ls /路径/Health\ Auto\ Export/Workout\ Data/HealthAutoExport-2026-02-22.json
```

### Q2: 成员名称可以包含中文吗？

可以，但文件名中的特殊字符会被替换为下划线。

### Q3: 如果某个成员的数据缺失会怎样？

脚本会跳过该成员并继续处理其他成员，输出中会显示错误信息。

### Q4: 可以只修改部分成员的路径吗？

可以，未修改的成员将使用默认路径。

### Q5: Workout路径可以和Health路径相同吗？

可以。有些用户的Health和Workout数据在同一个文件夹中，这样设置即可：

```python
MEMBER_2_HEALTH_DIR = Path.home() / "Health Data"
MEMBER_2_WORKOUT_DIR = Path.home() / "Health Data"  # 与Health相同
```
