# 健康报告邮件发送配置说明

## 概述

本文档说明如何配置健康报告自动邮件发送功能。

## 1. gog 邮件工具配置

### 1.1 检查 gog 是否已安装

```bash
gog --version
```

如果未安装，使用以下命令安装：
```bash
brew install openclaw/tap/gog
```

### 1.2 配置邮件账户

gog 需要配置 Gmail 或 Google Workspace 账户才能发送邮件。

#### 步骤1：创建 Google Cloud 项目
1. 访问 https://console.cloud.google.com/
2. 创建新项目或选择现有项目
3. 启用 Gmail API

#### 步骤2：配置 OAuth 凭据
1. 进入 "APIs & Services" > "Credentials"
2. 点击 "Create Credentials" > "OAuth client ID"
3. 选择 "Desktop app" 类型
4. 记下 Client ID 和 Client Secret

#### 步骤3：使用 gog 认证
```bash
gog auth login
```

按照提示输入 Client ID 和 Client Secret，完成 OAuth 授权流程。

### 1.3 验证邮件发送

```bash
echo "测试邮件内容" | gog send --to revolutionljk@gmail.com --subject "测试邮件"
```

## 2. 邮件发送脚本

### 脚本位置

```
scripts/send_health_report_email.py
```

### 用法

#### 方式1：指定具体文件
```bash
python3 scripts/send_health_report_email.py 2026-02-24 \
  /path/to/report1.pdf \
  /path/to/report2.pdf
```

#### 方式2：自动查找报告
```bash
python3 scripts/send_health_report_email.py 2026-02-24
```

脚本会自动在 `upload/` 目录查找指定日期的报告文件。

### 邮件内容

- **收件人**: revolutionljk@gmail.com
- **主题**: 健康日报 - {日期}
- **附件**: 所有生成的PDF报告
- **正文**: 包含报告列表和简要说明

## 3. 定时任务集成

定时任务 `daily-health-reports-v5` 已配置邮件发送步骤。

执行流程：
1. 生成所有报告（日报/周报/月报）
2. 调用邮件发送脚本
3. 在 Discord 通知中确认邮件发送状态

### 手动触发邮件发送

如果需要手动发送某天的报告：

```bash
cd /Users/jimmylu/.openclaw/workspace-health

# 先生成报告（如果有需要）
python3 scripts/generate_v5_medical_dashboard.py 2026-02-24 < ai_analysis.json

# 然后发送邮件
python3 scripts/send_health_report_email.py 2026-02-24
```

## 4. 故障排查

### 问题1：gog auth 失败

**症状**: `gog auth login` 报错或无法完成授权

**解决**:
1. 检查 Google Cloud 项目是否启用了 Gmail API
2. 确认 OAuth 凭据类型为 "Desktop app"
3. 检查是否已添加测试用户（在 OAuth consent screen 设置中）

### 问题2：邮件发送失败

**症状**: 脚本返回 "邮件发送失败"

**解决**:
1. 检查 gog 认证状态: `gog auth status`
2. 重新认证: `gog auth login`
3. 检查网络连接
4. 查看 gog 详细错误信息

### 问题3：附件过大

**症状**: 邮件发送失败，提示附件大小超过限制

**解决**:
- Gmail 限制附件总大小为 25MB
- 如果报告文件过大，可以考虑：
  - 压缩PDF文件
  - 使用云存储链接代替附件
  - 分批发送邮件

## 5. 配置文件

如果需要修改默认配置，可以创建配置文件：

```bash
mkdir -p ~/.config/health-reports
cat > ~/.config/health-reports/email.conf << 'EOF'
[email]
recipient = revolutionljk@gmail.com
subject_prefix = 健康日报
sender_name = Health Assistant
EOF
```

## 6. 测试邮件发送

在正式使用前，建议先进行测试：

```bash
# 创建一个测试PDF
echo "测试内容" > /tmp/test-report.pdf

# 发送测试邮件
python3 scripts/send_health_report_email.py $(date +%Y-%m-%d) /tmp/test-report.pdf
```

检查收件箱确认收到测试邮件。

---

**注意**: 邮件发送功能依赖 gog 工具和 Google 账户配置。请确保已完成上述配置步骤后再启用定时任务。
